The fix was simpler than expected.

- Solution

Add a SPA fallback in the `location` block of your Nginx config.

<CodeBlock>
```
location / {
    ...
    # SPA fallback
    proxy_intercept_errors on;
    error_page 404 = /index.html;
    # Serve index.html when the server thinks the path doesn't exist
}
```
</CodeBlock>

You donâ€™t need this fallback for a Next.js app. Why is that?

<br />

React apps are SPAs (Single Page Applications), so routing is handled on the client. All static assets are served up front, and subsequent page requests only fetch the data needed for updates. Because of this, when a different URL is requested, the server assumes the path doesnâ€™t existâ€”hence the Nginx 404. You need a fallback.

<br />
What about Next.js? Itâ€™s an SSR (Server Side Rendering) framework, where the server
handles routing and renders the requested path. So no separate fallback is needed.
If you do see a 404, you likely requested a page that truly doesnâ€™t exist.
