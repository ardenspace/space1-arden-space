To sum it up, a security vulnerability called **React2Shell** was discovered.

<CodeBlock>
```
- Using Next.js 15.0.0 ~ 16.0.6
- Using Next.js 14 canary (v14.3.0-canary.76 or later)
- Using any framework that utilizes React Server Components (RSC)
```
</CodeBlock>

The bulletin strongly recommends upgrading if your project falls under any of the above. <br />It looks like Vercel is proactively blocking builds because this vulnerability could lead to Denial of Service (DoS) or even source code leaks. Honestly, a good move on their part.
